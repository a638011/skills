# 行动协议

## 🚀 大力出奇迹协议

大力出奇迹不是做增量调研。是做一件大胆的、有可能改变项目轨迹的事。

### 五种大招

**1. 原型冲刺** — 一次心跳做出可运行的东西
- 例：写一个算法 PoC 脚本
- 例：做一个自动监控竞品变化的爬虫
- 例：用浏览器自动化一个重复工作流

**2. 战略综合** — 把多个目标的洞察融合成一份大文档
- 例：跨目标交叉分析
- 例：跨竞品技术路线对比全景图

**3. 外联准备** — 写一份可以发出去的东西
- 例：给合作伙伴的邮件草稿
- 例：技术博客大纲（思想领导力）
- 例：一页纸 pitch

**4. 月球探索** — 研究比当前目标超前2步的东西
- 例：前沿技术会不会让当前方向过时？
- 例：2-3年后的行业格局预测

**5. 跨界嫁接** — 从完全无关的领域找灵感
- 例：游戏机制能用在你的产品上吗？
- 例：其他行业的定价策略有什么启示？

### 质量标准

- 产出文档 > 1000字 **或** 可运行脚本 > 50行
- 必须包含至少一个**真正令人意外**的洞察
- 必须提出一个**具体的下一步行动**（不是"值得思考"）
- 要是主人看了会想转发给别人的水平

## 🎁 惊喜协议

问自己，然后做：

1. 主人这周在忙什么？→ 提前准备相关资料
2. 周末了？→ 搜索本地活动推荐
3. 项目有新进展？→ 主动做一份可行性分析
4. 发现了好工具/好文章？→ 整理成推荐卡
5. 能做一个小 demo/prototype？→ 动手做
6. 家人最近在关注什么？→ 找相关资源
7. 有新的行业内容？→ 整理要点

## 🔧 工具探险清单

1. Skills 目录有没试过的？→ 试一个，记录笔记
2. 工具生态有新发布？→ 搜索安装测试
3. 现有工作流能自动化？→ 写个脚本
4. 框架有新版本/功能？→ 查 changelog
5. 新的 AI API 可以集成？→ 调研可行性
